SQL & SQL Injection

SQL (structured query language) is a specialized programming language which was created to store and manipulate data within a relational database. Nearly all dynamic websites require a database of some sort, making SQL immensly popular worldwide. It is one of the most commonly used database languages on the web today.

Although SQL offers a very effective means of managing data within a database, it has its limitations. SQL injection is probably the most notable of these limitations. SQL injection is a vulnerability (design flaw) that allows unintended visitors to access a database. As indicated in the video above, this vulnerability is well documented, and there are many solutions that address this problem. Most of these precautions involve implementing some simple code to check for malicious queries. However, there is also software that can provide additional security.

Despite its dominance, SQL isnt the only option when it comes to databases. A relative newcomer (SQL was developed in the 70's) called NoSQL has been making progress recently. NoSQL (Not Only SQL) is a dabase management system that stores data differently than relational databases. Without getting into detail on the technical aspects, NoSQL is capable of operating faster and more efficiently than SQL in certain circumstances. They offer greater flexibility when doing queries, and they are more scalable. They are not necessarily a better solution, but rather more appropriate in certain circumstances.
