Classes and Objects

By Steve Ono, Posted on July 6, 2014

Object Orientation

In computer programming, there are many types of programming languages. I can't name the exact number, but there are somewhere in the order of a couple hundred languages out there. Perhaps one of the biggest paradigms that has altered computer programming has been the addition of object orientation. Programming languages that use object orientation are called object oriented programming languages, also known as OOP. In OOP, the most notable structures are classes and objects. They are specific to OOP, and they allow the programmer to dramatically reduce the amount of written code in a program.

Classes

Classes are structures that determine how data will be used. They are independent of any specific instances. The best way to describe them would be a blueprint of a townhouse. Since the houses in the neighborhood will pretty much have the same design, the blueprint will be one "master" design. When the individual houses are built, they may end up having different individual details, but the main design is the same. Classes are like blueprints, where objects are like the houses themselves.

Objects

Objects are instances of classes. Given the analogy of a blueprint, the objects are the actual houses. For example, one house may have yellow paint, while another one can have green paint. They are essentially the same design, but small details are different. When you instantiate an object from a class, you are filling that object with specific data, but that data is still structured within the design of the class.
